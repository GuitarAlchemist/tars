Makes self‑evolving behavior safe and cheap to try, instead of paralyzing you with “we must prove it’s perfect.”

Keeps agents decoupled.

2. Example FLUX Metascript Consuming Graphiti Output
Assume FLUX is a functional-ish DSL that transforms JSON-shaped structures into other structured data (e.g. grammar rules).

Input type (conceptual)
GraphitiSubgraph:

nodes: Node[]

edges: Edge[]

Output type
GrammarRules:

Let’s write pseudo‑FLUX:

metascript GraphitiToGrammar
  input GraphitiSubgraph
  output GrammarRules

  // Helper: find all functions with parameters
  def findFunctionPatterns(nodes, edges):
    let functions = filter(nodes, n -> n.type == "Function")
    in map(functions, f -> {
      let params =
        edges
        |> filter(e -> e.type == "HAS_PARAM" && e.from == f.id)
        |> map(e -> find(nodes, n -> n.id == e.to))

      {
        "functionNode": f,
        "parameters": params
      }
    })

  // Turn a function+params structure into a grammar rule
  def functionPatternToRule(pattern):
    let f = pattern.functionNode
    let params = pattern.parameters

    // Simple: only grammar for name + arity
    {
      "id": "rule:fsharp:function:" + f.name,
      "language": "fsharp",
      "kind": "function_declaration",
      "match": {
        "name": f.name,
        "arity": length(params)
      },
      "template": "let {{name}} {{#each params}}{{this.name}} {{/each}}= {{body}}"
    }

  def main(graph):
    let patterns = findFunctionPatterns(graph.nodes, graph.edges)
    let rules = map(patterns, p -> functionPatternToRule(p))

    { "rules": rules }
This script:

Walks Graphiti’s graph.

Finds Function nodes and their HAS_PARAM edges.

Produces grammar rules describing “what a function declaration looks like” for each one.

Later you generalize across many functions to produce more abstract rules (e.g., “any function with 2 parameters”).

0. Design Goals (What FLUX Is For)
FLUX is a pure(ish) transformation DSL used by TARS to:

Transform structured data → structured data
(JSON-like objects, lists, primitives)

Especially:

Graphiti graphs → grammar rules

ASTs → rewritten ASTs

Grammars → transformed grammars

Run in a sandboxed, deterministic way inside Cortex.

Key properties:

Side‑effect free inside a single run (no network, no IO).

Deterministic: same input, same output.

Strongly typed at the schema level, but implemented on top of JSON-like values.

Independent of any host language (can be interpreted from F#, Rust, Python, etc).

Think: “a tiny, functional, data-transformation language for graphs, grammars and ASTs.”

1. Core Concepts
1.1 Values
FLUX operates on JSON-like values:

null

bool

number (64-bit float, plus optional int hint)

string

list (ordered)

object (string → value map)

Plus typed wrappers:

Graph (object with nodes, edges)

Ast (language-specific tree representation)

Grammar (set of rules)

But these are just schemas: internal representation is still objects/lists.

1.2 Types
Types are structural and optional but recommended:

Any

Bool

Number

String

List<T>

Object (with optional field hints)

Alias types (e.g. type Node = Object { id: String, type: String, … })

Types in FLUX are used for:

Editor tooling / hints

Runtime validation (optional strict mode)

Evolution safety (TARS can reject metascripts that violate types)

2. Top-Level Structure
2.1 Metascript
A metascript is the compilation unit.

Basic shape:

metascript Name
  input  InputType
  output OutputType

  // optional type aliases
  type Node = Object {
    id: String,
    type: String
  }

  // functions
  fn helper(args...) -> ReturnType {
    // ...
  }

  fn main(input: InputType) -> OutputType {
    // ...
  }
end
Rules:

Exactly one metascript per file.

Exactly one main function (entry point).

input / output declare the shape expected by the host.

3. Syntax Overview
I’ll use a compact syntax borrowing from ML + JS.

3.1 Lexical
Identifiers: [A-Za-z_][A-Za-z0-9_]*

Strings: "double-quoted", supports \n, \", \\

Comments:

Single line: // comment

Block: /* comment */

Whitespace: ignored except as separator.

3.2 Types (Optional Annotations)
type Node = Object {
  id: String,
  type: String,
  language?: String
}

type Graph = Object {
  nodes: List<Node>,
  edges: List<Object>
}
? = optional field.

Types are advisory; runtime may ignore them unless strict mode is enabled.

4. Expressions & Statements
FLUX is expression‑oriented:

Blocks: { ... }

Local bindings: let

Conditionals: if ... then ... else ...

Function calls: f(x, y)

Lambdas: (x) -> expr

Pipeline: value |> f(...) (syntactic sugar)

4.1 Let Binding
let a = 1
let b = a + 41
4.2 Conditionals
let result =
  if count > 0 then
    "non-empty"
  else
    "empty"
4.3 Lists & Objects

}
4.4 Field Access & Updates
let lang = node.language

let node2 = node.with {
  language = "fsharp"
}
.with { ... } returns a shallow copy with updated fields.

7. Pattern Matching (v0.1 Light Version)
To keep FLUX small, we can have a limited pattern match expression:

match value with
  | { type: "Function", name: n } -> n
  | _ -> "unknown"
end
Semantics:

Patterns are objects with literal fields and binding fields.

_ is wildcard.

No nested patterns in v0.1 (can be added later).

This is really useful for:

discriminating node/edge types

consuming AST nodes

processing grammar rules

8. Grammar Representation
Grammar objects are regular FLUX objects with this schema (convention, not enforced):

type GrammarRule = Object {
  id: String,
  language: String,
  kind: String,           // e.g. "function_declaration"
  match: Object,          // structural conditions
  template: String,       // code template
  metadata?: Object       // arbitrary info
}

type Grammar = Object {
  rules: List<GrammarRule>
}
Example rule created in FLUX:

fn functionPatternToRule(pattern: Object) -> GrammarRule {
  let f = pattern.functionNode
  let params = pattern.parameters

  {
    id: "rule:fsharp:function:" + f.name,
    language: "fsharp",
    kind: "function_declaration",
    match: {
      name: f.name,
      arity: length(params)
    },
    template: "let {{name}} {{#each params}}{{this.name}} {{/each}}= {{body}}",
    metadata: {
      sourceNode: f.id
    }
  }
}
The templating syntax inside template (e.g. {{name}}) is not executed by FLUX itself; FLUX only constructs the rule object. A separate template engine (inside TARS) expands the template using context.

9. Example: Graphiti → Grammar Metascript
Putting it together into a full metascript for your earlier case:

metascript GraphitiToFSharpFunctionGrammar
  input  Graph      // { nodes: [...], edges: [...] }
  output Grammar    // { rules: [...] }

  type Node = Object {
    id: String,
    type: String,
    language?: String,
    name?: String
  }

  fn findFunctionPatterns(nodes: List<Node>, edges: List<Object>) -> List<Object> {
    let functions = filter(nodes, n -> n.type == "Function")

    map(functions, f -> {
      let params =
        edges
        |> filter(e -> e.type == "HAS_PARAM" && e.from == f.id)
        |> map(e -> find(nodes, n -> n.id == e.to))

      {
        functionNode: f,
        parameters: params
      }
    })
  }

  fn functionPatternToRule(pattern: Object) -> GrammarRule {
    let f = pattern.functionNode
    let params = pattern.parameters

    {
      id: "rule:fsharp:function:" + f.name,
      language: "fsharp",
      kind: "function_declaration",
      match: {
        language: "fsharp",
        arity: length(params)
      },
      template:
        "let {{name}} " +
        (params |> map(p -> p.name + " ") |> reduce("", (acc, s) -> acc + s)) +
        "= {{body}}",
      metadata: {
        sourceNode: f.id
      }
    }
  }

  fn main(graph: Graph) -> Grammar {
    let patterns = findFunctionPatterns(graph.nodes, graph.edges)
    let rules = map(patterns, p -> functionPatternToRule(p))
    { rules: rules }
  }
end

When Cortex runs a FLUX metascript:

Load spec:

TARS knows input and output schema from the header.

Validate input:

Coerce JSON from the calling context (Graphiti, AST, etc.) into the expected “shape”.

Interpret:

Execute main(input) in a sandbox (no IO, no external calls).

Validate output:

Ensure it matches the declared output schema (or at least the top‑level shape).

Return structured result back to Cortex:

Usually: Grammar, Ast, or some other object that will drive another TARS module.

In evolution terms, FLUX metascripts themselves are:

Versioned artifacts (flux://GraphitiToFSharpFunctionGrammar@1.3.0)

Subject to fitness scoring & tests like any other component.

Custom higher-order combinators beyond map, filter, reduce.

You can add features only when a real metascript needs them.

type FunctionImpl = FluxValue list -> FluxValue

/// Runtime environment:
/// - variables (for let-bound values)
/// - functions (either user-defined or built-in)
type FluxEnv =
    { Vars      : Map<string, FluxValue>
      Functions : Map<string, FunctionImpl> }

module FluxEnv =
    let empty =
        { Vars = Map.empty
          Functions = Map.empty }

    let withVar name value env =
        { env with Vars = env.Vars |> Map.add name value }

    let withFunction name impl env =
        { env with Functions = env.Functions |> Map.add name impl }

    let tryGetVar name env =
        env.Vars |> Map.tryFind name

    let tryGetFunction name env =
        env.Functions |> Map.tryFind name

3.3 Interpreter
src/Tars.Flux/FluxInterpreter.fs:

namespace Tars.Flux

open FluxValue

module FluxInterpreter =

    let rec eval (env:FluxEnv) (expr:Expr) : FluxValue =
        match expr with
        | EValue v -> v

        | EVar name ->
            match FluxEnv.tryGetVar name env with
            | Some v -> v
            | None -> failwithf "Unbound variable: %s" name

        | ELet (name, rhs, body) ->
            let v = eval env rhs
            let env' = FluxEnv.withVar name v env
            eval env' body

        | EIf (cond, tBranch, fBranch) ->
            match eval env cond with
            | FBool true -> eval env tBranch
            | FBool false -> eval env fBranch
            | v -> failwithf "If condition not a bool: %A" v

        | EBinOp (op, a, b) ->
            let av = eval env a
            let bv = eval env b
            evalBinOp op av bv

        | ECall (name, args) ->
            let argVals = args |> List.map (eval env)
            match FluxEnv.tryGetFunction name env with
            | Some f -> f argVals
            | None -> failwithf "Unknown function: %s" name

        | ELambda _ ->
            // v0.1 you can either:
            //  - reject user lambdas
            //  - or capture them as a special FluxValue form
            failwith "User lambdas not implemented yet"

        | EFieldGet (objExpr, field) ->
            let objVal = eval env objExpr
            match objVal with
            | FObject m ->
                Map.tryFind field m |> Option.defaultValue FNull
            | _ -> failwith "Field access on non-object"

        | EWithObject (objExpr, updates) ->
            let baseVal = eval env objExpr
            let updated =
                match baseVal with
                | FObject m ->
                    updates
                    |> List.fold (fun acc (key, ex) ->
                        let v = eval env ex
                        Map.add key v acc
                    ) m
                    |> FObject
                | _ -> failwith "with { } on non-object"
            updated

        | EList exprs ->
            exprs |> List.map (eval env) |> FList

        | EObject fields ->
            fields
            |> List.map (fun (k, ex) -> k, eval env ex)
            |> FluxValue.ofObject

        | EPipe (valueExpr, fnName, moreArgs) ->
            let v = eval env valueExpr
            let args = v :: (moreArgs |> List.map (eval env))
            match FluxEnv.tryGetFunction fnName env with
            | Some f -> f args
            | None -> failwithf "Unknown function in pipe: %s" fnName

        | EMatch (valueExpr, cases) ->
            let v = eval env valueExpr
            evalMatch env v cases

    and evalBinOp op a b =
        match op, a, b with
        | Add, FNumber x, FNumber y -> FNumber (x + y)
        | Sub, FNumber x, FNumber y -> FNumber (x - y)
        | Mul, FNumber x, FNumber y -> FNumber (x * y)
        | Div, FNumber x, FNumber y -> FNumber (x / y)
        | Eq,  x, y -> FBool (x = y)
        | Ne,  x, y -> FBool (x <> y)
        | _ -> failwithf "Unsupported binary op: %A %A %A" op a b

    and evalMatch (env:FluxEnv) (value:FluxValue) (cases:MatchCase list) : FluxValue =
        let rec loop = function
            | [] -> failwith "Non-exhaustive match"
            | (pat, body)::rest ->
                match tryMatchPattern value pat with
                | Some bindings ->
                    let env' =
                        bindings
                        |> List.fold (fun e (name, v) ->
                            FluxEnv.withVar name v e
                        ) env
                    eval env' body
                | None ->
                    loop rest
        loop cases

    and tryMatchPattern (value:FluxValue) (pat:Pattern) : (string * FluxValue) list option =
        match pat with
        | PWildcard -> Some []
        | PObject fields ->
            match value with
            | FObject m ->
                let rec matchFields bindings remaining =
                    match remaining with
                    | [] -> Some bindings
                    | (fieldName, subPat)::rest ->
                        match Map.tryFind fieldName m with
                        | None -> None
                        | Some fieldValue ->
                            match tryMatchPattern fieldValue subPat with
                            | None -> None
                            | Some bs -> matchFields (bindings @ bs) rest
                matchFields [] fields
            | _ -> None

3. Evolution Rate Controller
Controls how often L1/L2 can mutate, based on layer & metrics.

File: src/Tars.Core/EvolutionRateController.fs

namespace Tars.Core

open System

type EvolutionDecision =
    | AllowMutation
    | DeferMutation of reason : string
    | RejectMutation of reason : string

/// Global configuration for how aggressive each layer can mutate.
[<CLIMutable>]
type EvolutionRateConfig =
    { L1_MinSuccessfulRunsBeforeMutation : int64
      L1_MaxMutationsPerDay              : int64
      L2_MinDaysBetweenPolicyChanges     : int
      L2_MaxPolicyMutationsPerQuarter    : int }

module EvolutionRateConfig =
    let defaultConfig =
        { L1_MinSuccessfulRunsBeforeMutation = 50L
          L1_MaxMutationsPerDay              = 10L
          L2_MinDaysBetweenPolicyChanges     = 7
          L2_MaxPolicyMutationsPerQuarter    = 3 }

type IEvolutionRateController =
    abstract member ShouldMutate :
        component : ILayeredComponent *
        metrics   : LayerMetrics *
        config    : EvolutionRateConfig *
        now       : DateTimeOffset
            -> EvolutionDecision

/// Simple reference implementation.
type DefaultEvolutionRateController() =
    interface IEvolutionRateController with
        member _.ShouldMutate (component, metrics, config, now) =
            match component.LayerTag.Layer with
            | L0_Fast ->
                // L0 never mutates structure; it's ephemeral.
                RejectMutation "L0 components are not structurally mutated."

            | L1_Mid ->
                if metrics.SuccessfulRuns < config.L1_MinSuccessfulRunsBeforeMutation then
                    DeferMutation $"Need at least %d{config.L1_MinSuccessfulRunsBeforeMutation} successful runs before mutation."
                else
                    // very naive daily cap based on CreatedAt / LastUpdated.
                    let daysSinceCreation =
                        (now - metrics.CreatedAt).TotalDays |> int
                    let approxMutationsPerDay =
                        if daysSinceCreation <= 0 then metrics.MutationsApplied
                        else metrics.MutationsApplied / int64 daysSinceCreation

                    if approxMutationsPerDay >= config.L1_MaxMutationsPerDay then
                        DeferMutation "Daily mutation rate exceeded."
                    else
                        AllowMutation

            | L2_Slow ->
                let daysSinceLastUpdate =
                    (now - metrics.LastUpdatedAt).TotalDays |> int
                if daysSinceLastUpdate < config.L2_MinDaysBetweenPolicyChanges then
                    DeferMutation $"L2 policy changes too frequent; wait %d{config.L2_MinDaysBetweenPolicyChanges - daysSinceLastUpdate} more day(s)."
                else
                    AllowMutation

            | L3_Rare ->
                RejectMutation "L3 changes require human approval."
You can swap in a more sophisticated controller later; this is just the starting skeleton.

        member _.Metrics = state.Metrics

module LayerTag =
    let L0 = { Layer = L0_Fast; Timescale = Fast }
    let L1 = { Layer = L1_Mid;  Timescale = Mid }
    let L2 = { Layer = L2_Slow; Timescale = Slow }
    let L3 = { Layer = L3_Rare; Timescale = Rare }

[<CLIMutable>]
type LayerMetrics =
    { ComponentId      : string
      LayerTag         : LayerTag
      Version          : string
      CreatedAt        : DateTimeOffset
      LastUpdatedAt    : DateTimeOffset
      SuccessfulRuns   : int64
      FailedRuns       : int64
      MutationsApplied : int64 }

module LayerMetrics =
    let init componentId version tag now =
        { ComponentId      = componentId
          LayerTag         = tag
          Version          = version
          CreatedAt        = now
          LastUpdatedAt    = now
          SuccessfulRuns   = 0L
          FailedRuns       = 0L
          MutationsApplied = 0L }

    let recordSuccess now m =
        { m with SuccessfulRuns = m.SuccessfulRuns + 1L
                 LastUpdatedAt  = now }

    let recordFailure now m =
        { m with FailedRuns    = m.FailedRuns + 1L
                 LastUpdatedAt = now }

    let recordMutation now m =
        { m with MutationsApplied = m.MutationsApplied + 1L
                 LastUpdatedAt    = now }

namespace Tars.Flux

type FluxValue =
    | FNull
    | FBool   of bool
    | FNumber of float
    | FString of string
    | FList   of FluxValue list
    | FObject of Map<string, FluxValue>

module FluxValue =
    let ofBool b = FBool b
    let ofInt (i:int) = FNumber (float i)
    let ofFloat f = FNumber f
    let ofString s = FString s
    let ofList xs = FList xs
    let ofObject kvs = kvs |> Map.ofList |> FObject

    let tryGetField key = function
        | FObject m -> Map.tryFind key m
        | _         -> None

    let withField key value = function
        | FObject m -> FObject (Map.add key value m)
        | _         -> failwith "withField: not an object"

    let asList = function
        | FList xs -> xs
        | _ -> failwith "Expected list"

    let asObject = function
        | FObject m -> m
        | _ -> failwith "Expected object"

    let asString = function
        | FString s -> s
        | _ -> failwith "Expected string"

    let asNumber = function
        | FNumber n -> n
        | _ -> failwith "Expected number"

    { Vars      : Map<string, FluxValue>
      Functions : Map<string, FunctionImpl> }

module FluxEnv =
    let empty =
        { Vars = Map.empty
          Functions = Map.empty }

    let withVar name v env =
        { env with Vars = env.Vars |> Map.add name v }

    let withFunction name impl env =

src/Tars.Flux/FluxInterpreter.fs
namespace Tars.Flux

open FluxValue

module FluxInterpreter =

    type EvalError = string

    let rec eval env expr : FluxValue =
        match expr with
        | EValue v -> v
        | EVar name ->
            match FluxEnv.tryGetVar name env with
            | Some v -> v
            | None -> failwithf "Unbound variable: %s" name
        | ELet (name, rhs, body) ->
            let v = eval env rhs
            eval (FluxEnv.withVar name v env) body
        | EIf (c, t, f) ->
            match eval env c with
            | FBool true  -> eval env t
            | FBool false -> eval env f
            | v -> failwithf "If condition not bool: %A" v
        | EBinOp (op, a, b) ->
            let av, bv = eval env a, eval env b
            match op, av, bv with
            | Add, FNumber x, FNumber y -> FNumber (x + y)
            | Sub, FNumber x, FNumber y -> FNumber (x - y)
            | Mul, FNumber x, FNumber y -> FNumber (x * y)
            | Div, FNumber x, FNumber y -> FNumber (x / y)
            | Eq,  x, y -> FBool (x = y)
            | Ne,  x, y -> FBool (x <> y)
            | _ -> failwithf "Unsupported binop: %A %A %A" op av bv
        | ECall (name, args) ->
            let argVals = args |> List.map (eval env)
            match FluxEnv.tryGetFunction name env with
            | Some f -> f argVals
            | None   -> failwithf "Unknown function: %s" name
        | EList xs ->
            xs |> List.map (eval env) |> FList
        | EObject fields ->
            fields
            |> List.map (fun (k, e) -> k, eval env e)
            |> FluxValue.ofObject
        | EFieldGet (objEx, field) ->
            match eval env objEx with
            | FObject m -> Map.tryFind field m |> Option.defaultValue FNull
            | _ -> failwith "Field access on non-object"
        | EWithObject (objEx, updates) ->
            let baseVal = eval env objEx
            match baseVal with
            | FObject m ->
                updates
                |> List.fold (fun acc (k, ex) ->
                    Map.add k (eval env ex) acc) m
                |> FObject
            | _ -> failwith "with { } on non-object"

type FluxExecutionError =
    | MetascriptNotFound of string
    | RuntimeError of string

type IFluxStore =
    abstract GetMetascript : string -> Metascript option

type FluxEngine(store: IFluxStore) =

    member _.Execute(metascriptId:string, input:FluxValue) =
        match store.GetMetascript metascriptId with
        | None -> Error (MetascriptNotFound metascriptId)
        | Some _ms ->
            try
                // v0.1: just call "main" builtin with input
                let env =
                    FluxEnv.empty
                    |> List.fold (fun e (n, impl) ->
                        FluxEnv.withFunction n impl e) FluxStdLib.builtins

                        match args with
                        | [v] -> v
                        | _ -> failwith "main: expected 1 arg")

                    FluxInterpreter.eval env (Expr.ECall ("main", [Expr.EValue input]))
                Ok result
            with ex ->
                Error (RuntimeError ex.Message)

1. Kernel.fs — Agents + Supervision Skeleton
File: src/Tars.Core/Kernel.fs

This gives you:

Erlang‑style behaviour interface

Agent host using MailboxProcessor

A tiny supervision model (restart/stop)

namespace Tars.Core

open System

type AgentId = string

/// Messages that flow between agents.
/// V0: keep it open; you’ll likely replace Payload with a DU later.
[<CLIMutable>]
type AgentMessage<'payload> =
    { From    : AgentId option
      To      : AgentId
      Payload : 'payload }

/// Result of handling a single message.
type AgentStepResult<'state> =
    | Continue of 'state
    | Stop of 'state
    | Crash of exn * 'state option

/// Behaviour contract for an agent (like Erlang's gen_server/gen_statem).
type IAgentBehaviour<'state, 'msg> =
    /// Initialize state. Called once at agent start.
    abstract member Init :
        id      : AgentId *
        now     : DateTimeOffset
            -> 'state

    /// Handle an incoming message for this agent.
    abstract member Handle :
        state   : 'state *
        message : AgentMessage<'msg> *
        now     : DateTimeOffset
            -> AgentStepResult<'state>

    /// Called when the agent is shutting down (normal or crash).
    abstract member Terminate :
        state   : 'state option *
        reason  : string *
        now     : DateTimeOffset
            -> unit

/// Supervision strategy for a child agent.
type SupervisorStrategy =
    | Restart
    | Stop
    | Escalate

/// A descriptor for a supervised agent.
[<CLIMutable>]
type AgentSpec<'state, 'msg> =
    { Id        : AgentId
      Behaviour : IAgentBehaviour<'state, 'msg>
      Strategy  : SupervisorStrategy }

/// Internal representation of a running agent.
type internal AgentRuntime<'state, 'msg> =
    { Id        : AgentId
      Behaviour : IAgentBehaviour<'state, 'msg>
      Mailbox   : MailboxProcessor<AgentMessage<'msg>>
      mutable State : 'state }

/// Kernel = registry of agents + supervision.
/// V0: super simple; you can evolve to trees later.
type Kernel<'msg>() =

    let mutable agents : Map<AgentId, obj> = Map.empty

    let startAgent (spec: AgentSpec<'state,'msg>) =
        let initialState =
            spec.Behaviour.Init(spec.Id, DateTimeOffset.UtcNow)

        let rec loop (runtime: AgentRuntime<'state,'msg>) (inbox: MailboxProcessor<AgentMessage<'msg>>) =
            async {
                let! msg = inbox.Receive()
                let now = DateTimeOffset.UtcNow
                let result =
                    try
                        runtime.Behaviour.Handle(runtime.State, msg, now)
                    with ex ->
                        Crash (ex, Some runtime.State)

                match result with
                | Continue newState ->
                    runtime.State <- newState
                    return! loop runtime inbox

                | Stop newState ->
                    runtime.State <- newState
                    runtime.Behaviour.Terminate(Some newState, "normal", now)

                | Crash (ex, optState) ->
                    runtime.Behaviour.Terminate(optState, $"crash: {ex.Message}", now)
                    match spec.Strategy with
                    | Restart ->
                        // naive restart: new state, same agent id.
                        let restartedState = spec.Behaviour.Init(spec.Id, now)
                        runtime.State <- restartedState
                        return! loop runtime inbox
                    | Stop ->
                        () // Mailbox exits, agent removed by supervisor externally.
                    | Escalate ->
                        // For now, treat as Stop; later, bubble up to higher supervisor.
                        ()
            }

        let mailbox =
            MailboxProcessor.Start(fun inbox ->
                let runtime =
                    { Id        = spec.Id
                      Behaviour = spec.Behaviour
                      Mailbox   = inbox
                      State     = initialState }
                loop runtime inbox)

        let runtimeObj : obj = box { Id = spec.Id; Behaviour = spec.Behaviour; Mailbox = mailbox; State = initialState }
        agents <- Map.add spec.Id runtimeObj agents

    member _.Spawn<'state,'msg> (spec: AgentSpec<'state,'msg>) =
        startAgent spec

    member _.Post<'msg> (msg: AgentMessage<'msg>) =
        match Map.tryFind msg.To agents with
        | None -> failwithf "No agent with id '%s'." msg.To
        | Some runtimeObj ->
            let r = runtimeObj :?> AgentRuntime<obj,'msg> // NOTE: you’ll want a stronger type story later
            r.Mailbox.Post msg

/// Example echo behaviour (for testing).
type EchoBehaviour<'msg>() =
    interface IAgentBehaviour<unit, 'msg> with
        member _.Init (id, now) =
            printfn "[%O] EchoAgent %s starting." now id
            ()

        member _.Handle (state, msg, now) =
            printfn "[%O] EchoAgent %s got: %A" now msg.To msg.Payload
            Continue state

        member _.Terminate (state, reason, now) =
            printfn "[%O] EchoAgent terminating: %s" now reason
This is intentionally simple; you’ll probably want to:

specialise the types ('msg as a DU),

improve safety around obj casts,

add supervision trees, etc.

2. Tars.Skills.Graphiti — Protocol + Client Stub
Here we translate the Graphiti skill manifest/protocol into F# types and give you a client interface you can hook up later (e.g., to a Python process).

File: src/Tars.Skills/GraphitiSkill.fs

namespace Tars.Skills

open System
open System.Text.Json
open System.Text.Json.Serialization

/// Graphiti skill operations
[<RequireQualifiedAccess>]
type GraphitiOp =
    | IngestEpisodes
    | QuerySubgraph
    | ExtractCodePatterns

module GraphitiOp =
    let toString = function
        | GraphitiOp.IngestEpisodes    -> "ingest_episodes"
        | GraphitiOp.QuerySubgraph    -> "query_subgraph"
        | GraphitiOp.ExtractCodePatterns -> "extract_code_patterns"

    let ofString = function
        | "ingest_episodes"     -> GraphitiOp.IngestEpisodes
        | "query_subgraph"      -> GraphitiOp.QuerySubgraph
        | "extract_code_patterns" -> GraphitiOp.ExtractCodePatterns
        | s -> failwithf "Unknown Graphiti op: %s" s

[<CLIMutable>]
type GraphitiEpisode =
    { [<JsonPropertyName("episode_id")>]
      EpisodeId : string

      [<JsonPropertyName("type")>]
      Type : string  // "code" | "text" | ...

      [<JsonPropertyName("language")>]
      Language : string

      [<JsonPropertyName("path")>]
      Path : string

      [<JsonPropertyName("content")>]
      Content : string

      [<JsonPropertyName("metadata")>]
      Metadata : Map<string,obj> }

[<CLIMutable>]
type GraphitiSubgraphFilter =
    { [<JsonPropertyName("episode_ids")>]
      EpisodeIds : string array option

      [<JsonPropertyName("entity_types")>]
      EntityTypes : string array option

      [<JsonPropertyName("max_hops")>]
      MaxHops : int option }

[<CLIMutable>]
type GraphitiNode =
    { [<JsonPropertyName("id")>]
      Id : string

      [<JsonPropertyName("type")>]
      Type : string

      [<JsonPropertyName("language")>]
      Language : string option

      [<JsonPropertyName("name")>]
      Name : string option

      [<JsonPropertyName("metadata")>]
      Metadata : Map<string,obj> option }

[<CLIMutable>]
type GraphitiEdge =
    { [<JsonPropertyName("id")>]
      Id : string

      [<JsonPropertyName("type")>]
      Type : string

      [<JsonPropertyName("from")>]
      From : string

      [<JsonPropertyName("to")>]
      To : string }

[<CLIMutable>]
type GraphitiSubgraph =
    { [<JsonPropertyName("nodes")>]
      Nodes : GraphitiNode array

      [<JsonPropertyName("edges")>]
      Edges : GraphitiEdge array }

[<CLIMutable>]
type GraphitiPattern =
    { [<JsonPropertyName("id")>]
      Id : string

      [<JsonPropertyName("description")>]
      Description : string

      [<JsonPropertyName("example")>]
      Example : string

      [<JsonPropertyName("graph_signature")>]
      GraphSignature : Map<string,obj> }

[<CLIMutable>]
type GraphitiSkillRequest =
    { [<JsonPropertyName("request_id")>]
      RequestId : string

      [<JsonPropertyName("op")>]
      Op : string

      [<JsonPropertyName("payload")>]
      Payload : JsonElement }

[<CLIMutable>]
type GraphitiSkillError =
    { [<JsonPropertyName("code")>]
      Code : string

      [<JsonPropertyName("message")>]
      Message : string }

[<CLIMutable>]
type GraphitiSkillResponse =
    { [<JsonPropertyName("request_id")>]
      RequestId : string

      [<JsonPropertyName("status")>]
      Status : string

      [<JsonPropertyName("error")>]
      Error : GraphitiSkillError option

      [<JsonPropertyName("result")>]
      Result : JsonElement option }

/// High-level client you can implement using stdio, HTTP, or whatever.
type IGraphitiClient =
    abstract member IngestEpisodes :
        groupId : string *
        episodes : GraphitiEpisode array
            -> Async<Result<int, string>>   // returns #ingested or error

    abstract member QuerySubgraph :
        groupId : string *
        filter  : GraphitiSubgraphFilter
            -> Async<Result<GraphitiSubgraph, string>>

    abstract member ExtractCodePatterns :
        groupId : string *
        language : string *
        modules  : string array
            -> Async<Result<GraphitiPattern array, string>>

3. GraphitiToFSharpGrammar — Concrete FLUX Metascript as F# Data
This is a real Metascript value in your existing Tars.Flux AST, representing:

metascript GraphitiToFSharpFunctionGrammar
  input  Graph
  output Grammar

  fn main(graph) {
    // For v0.1, return an empty grammar: { rules: [] }
  }
end
You can make it more complex as your interpreter grows (map/filter, pattern matching, etc.).

File: src/Tars.Flux/GraphitiToFSharpGrammar.fs

namespace Tars.Flux

open FluxValue

/// A minimal Graphiti → Grammar FLUX metascript encoded in F#.
/// V0.1: just returns { rules = [] }.
/// Later you can flesh this out with real pattern extraction.
module GraphitiToFSharpGrammar =

    let metascript : Metascript =
        let mainBody =
            // Equivalent to: { rules: [] }
            Expr.EObject
                [ "rules", Expr.EList [] ]

        let mainFn =
            { Name   = "main"
              Params = [ { Name = "graph" } ] // currently unused
              Body   = mainBody }

        { Name       = "GraphitiToFSharpFunctionGrammar"
          InputType  = Some "Graph"      // advisory only; you’ll enforce in Cortex
          OutputType = Some "Grammar"    // advisory
          Functions  = [ mainFn ] }
Then, in your FluxEngine, you can:

store this metascript in a simple in‑memory IFluxStore implementation,

test-call it with a dummy Graph-shaped FluxValue.

Example in Tars.Cortex (very small in‑memory store):

File: src/Tars.Cortex/FluxStore.fs

namespace Tars.Cortex

open System.Collections.Generic
open Tars.Flux

type InMemoryFluxStore(initial : seq<string * Metascript>) =
    let dict = Dictionary<string, Metascript>()
    do
        for (id, ms) in initial do
            dict.[id] <- ms

    interface IFluxStore with
        member _.GetMetascript id =
            match dict.TryGetValue id with
            | true, ms -> Some ms
            | _ -> None
And wiring it up somewhere:

open Tars.Flux.GraphitiToFSharpGrammar

let store =
    InMemoryFluxStore(
        seq {
            "flux://GraphitiToFSharpFunctionGrammar@0.1.0", metascript
        })

let engine = FluxEngine(store)

let dummyGraph =
    // { nodes: []; edges: [] } as FluxValue
    FluxValue.ofObject
        [ "nodes", FluxValue.ofList []
          "edges", FluxValue.ofList [] ]

let result = engine.Execute("flux://GraphitiToFSharpFunctionGrammar@0.1.0", dummyGraph)
// Expect: Ok (FObject { "rules" = FList [] })
