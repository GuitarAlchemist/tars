# TARS Self-Extension Architecture

## Vision

> **"Metascripts are the safe sandbox where TARS experiments with new capabilities before they become permanent."**

TARS extends itself not by modifying core F# code, but by:
1. **Writing new metascripts** - Workflows that orchestrate existing tools
2. **Creating new tools** - Registered dynamically without recompilation
3. **Defining new grammars** - Constrain LLM outputs to valid syntax
4. **Composing capabilities** - Building complex behaviors from simple primitives

## Extension Mechanisms

### 1. Metascript-Based Extension

TARS can write new metascripts to extend its capabilities:

```tars
meta {
    name = "analyze_code_patterns"
    description = "Custom tool created by TARS for pattern detection"
    author = "TARS-self-improvement"
    version = "1.0"
}

FSHARP(output="patterns") {
    // Detect common patterns in code
    let code = System.IO.File.ReadAllText("${input_file}")
    let patterns = 
        [
            if code.Contains("| _ -> ()") then yield "catch-all-silent"
            if code.Contains("mutable") then yield "mutable-state"
            if code.Contains("Async.RunSynchronously") then yield "sync-over-async"
        ]
    patterns |> String.concat ", "
}

QUERY(output="suggestions", grammar="RefactoringSuggestions") {
    Given these patterns found in the code: ${patterns}
    
    Suggest specific refactoring actions to improve the code quality.
}
```

### 2. Dynamic Tool Registration

TARS can create new tools at runtime:

```fsharp
// TARS generates this tool definition
let newTool = {
    Name = "check_invariant"
    Description = "Checks if a code invariant holds"
    Version = "1.0.0"
    Execute = fun input -> async {
        // Tool logic generated by TARS
        let result = validateInvariant input
        return Ok result
    }
}

// Register without recompilation
toolRegistry.Register(newTool)
```

### 3. Grammar Evolution

TARS can define new grammars to constrain LLM outputs:

```ebnf
// RefactoringSuggestions.ebnf - Created by TARS
suggestions ::= suggestion+
suggestion  ::= "- " action ": " description "\n"
action      ::= "EXTRACT" | "INLINE" | "RENAME" | "REMOVE" | "DOCUMENT"
description ::= [^\n]+
```

## Safety Boundaries

### What TARS CAN do:
- Create/modify metascripts in `/metascripts/generated/`
- Register new tools with the ToolRegistry
- Define new grammars in `/grammars/generated/`
- Compose existing tools into new workflows

### What TARS CANNOT do:
- Modify core F# source files (without Plan validation)
- Change the HybridBrain execution engine
- Bypass the symbolic validation layer
- Execute unvalidated plans

## Implementation: Self-Extension Service

```fsharp
type SelfExtensionService = {
    /// Generate a new metascript based on a capability description
    CreateMetascript: string -> Task<Result<Metascript, string>>
    
    /// Register a new dynamic tool
    CreateTool: ToolDefinition -> Task<Result<unit, string>>
    
    /// Define a new grammar
    CreateGrammar: GrammarDefinition -> Task<Result<unit, string>>
    
    /// Evolve an existing metascript based on feedback
    EvolveMetascript: Metascript * Feedback -> Task<Result<Metascript, string>>
    
    /// List all self-generated extensions
    ListExtensions: unit -> ExtensionManifest
}
```

## Example: TARS Creates a New Capability

**User**: "TARS, create a tool that analyzes my git history for code quality trends"

**TARS**:
1. **Understands** the request (LLM)
2. **Plans** a new metascript (HybridBrain)
3. **Validates** the plan (Symbolic layer)
4. **Creates** the metascript:

```tars
meta {
    name = "git_quality_analyzer"
    author = "TARS"
    version = "1.0"
}

COMMAND(output="commits") {
    git log --oneline -n 50
}

FSHARP(output="commit_count") {
    "${commits}".Split('\n').Length
}

QUERY(output="analysis") {
    Analyze these ${commit_count} commits and identify:
    1. Code churn patterns
    2. Most modified files
    3. Quality improvement opportunities
    
    Commits:
    ${commits}
}
```

5. **Registers** the metascript
6. **Executes** on demand
